**Pattern reference (from RiveRenderer.ts):**
- LOOK_LERP_FACTOR = 0.1 (平滑系数)
- 归一化映射: `(canvasPixel / canvasSize) * 2 - 1` → -1.0~1.0
- 削波到 [-1, 1]
- 仅在变化 > 0.001 时更新参数（避免冗余调用）

**Normalization logic (from PetStage.tsx lines 226-229):**

```typescript
// canvasX/canvasY are pixel values relative to canvas element
// These get passed directly to lookAt(canvasX, canvasY)
// Inside lookAt, normalize to -1..1 range using canvas dimensions
```

**Important: ParamAngleX/ParamAngleY** in Live2D are typically in degrees, not normalized -1..1.
- The range for ParamAngleX is typically -30 to +30 degrees (left-right head turn)
- The range for ParamAngleY is typically -30 to +30 degrees (up-down head tilt)
- Map: normalized mouse (-1..1) × 30 → degrees

But different models may have different ranges. Follow the convention of using degrees with ±30 range, which is the Cubism standard.

    ```typescript
    /** 每帧调用: lerp 平滑并设置 ParamAngleX/ParamAngleY */
    private updateMouseFollow(): void {
      // 指数平滑 (lerp)
      this.currentLookX += (this.targetLookX - this.currentLookX) * this.LOOK_LERP_FACTOR;
      this.currentLookY += (this.targetLookY - this.currentLookY) * this.LOOK_LERP_FACTOR;

      const model = this.userModel?.getModel();
      if (!model) return;

      // Live2D ParamAngle 通常用度数表示，范围 -30~+30
      // 映射: normalized (-1..1) × 30 → degrees
      const angleX = this.currentLookX * 30;
      const angleY = this.currentLookY * 30;

      // 仅在变化显著时更新（避免冗余 WASM 调用）
      if (this.cachedAngleXIndex >= 0 && Math.abs(this.currentLookX - this.lastLookX) > 0.001) {
        model.setParameterValueByIndex(this.cachedAngleXIndex, angleX);
        this.lastLookX = this.currentLookX;
      }
      if (this.cachedAngleYIndex >= 0 && Math.abs(this.currentLookY - this.lastLookY) > 0.001) {
        model.setParameterValueByIndex(this.cachedAngleYIndex, angleY);
        this.lastLookY = this.currentLookY;
      }
    }
    ```

    private startLoop(): void {
      const loop = () => {
