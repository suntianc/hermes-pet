    ```typescript
    export type ModelType = 'rive' | 'live2d';
    ```

    2. model-registry.ts - 向 ModelConfig 添加 Live2D 特定字段:
    - type 字段默认为 'live2d'（对于 Rive 模型保持 'rive' 兼容）
    - 添加 live2d 元数据块（可选）

    ```typescript
    export interface ModelConfig {
      id: string;
      name: string;
      path: string;
      type?: ModelType;
      window?: { width: number; height: number };
      canvas?: { width: number; height: number };
      actions?: Record<string, ModelActionConfig>;
      capabilities?: ModelCapabilities;
      /** Live2D 模型特定元数据（可选，运行时从 .model3.json 自动解析） */
      live2d?: Live2DMetadata;
    }

    export interface Live2DMetadata {
      /** .model3.json 配置中定义的 Motion 组 */
      motions?: string[];
      /** .model3.json 配置中定义的表情 */
      expressions?: string[];
      /** .model3.json 配置中定义的物理效果文件 */
      physics?: string;
      /** .model3.json 配置中定义的姿势文件 */
      pose?: string;
      /** 模型显示信息 */
      displayInfo?: {
        width: number;
        height: number;
        xOrigin: number;
        yOrigin: number;
      };
    }
    ```

    3. model-registry.ts - 在 loadModelConfigs 中添加对 type 字段的映射:
    确保从 Rust 后端 model_list 返回的 ModelConfigDTO 中的 type 字段正确传递。
    当前代码已通过展开赋值（spread）传递所有字段，包括 type。
    
    4. tauri-types.ts - 更新 ModelConfigDTO 注释:
    添加注释说明 type 字段现在是 'rive' | 'live2d' 字符串。
    ModelConfigDTO 的 type 字段已存在（type?: string），无需修改接口定义。
