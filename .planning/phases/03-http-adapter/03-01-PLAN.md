<objective>
Create the embedded axum HTTP adapter on port 18765 with graceful shutdown wiring.

**Purpose:** Replace the Electron HTTP adapter (port 18765, two endpoints) with a Rust axum server running on Tauri's shared Tokio runtime. External agents (AI coding tools, etc.) POST agent events to `/adapter` and the server normalizes + forwards them to the Tauri WebView as `pet:event` events.

**Scope boundary:** Rust backend only — axum server, CancellationToken graceful shutdown, event types and normalization. Frontend event listeners remain unchanged (Phase 6 handles frontend IPC migration).

**Key decisions:**
- Axum via `tauri::async_runtime::spawn()` on shared Tokio runtime
- CancellationToken for graceful shutdown (from `tokio-util` "sync" feature)
- Two endpoints: POST /adapter, GET /adapter/capabilities
- CORS headers for cross-origin agent integration
- Port configurable via `VIVIPET_ADAPTER_PORT` env var (default 18765)
- Store CancellationToken in AppState for lifecycle-driven shutdown
- Convert lib.rs from `.run()` convenience to `.build().run()` pattern for RunEvent::Exit handling
</objective>

  <name>Task 4: Create adapter/lifecycle.rs — CancellationToken shutdown</name>
  <files>apps/desktop/src-tauri/src/adapter/lifecycle.rs</files>
  <action>
    Create a thin lifecycle management module for the adapter server.

    ```rust
    use tokio_util::sync::CancellationToken;

    /// Manages the adapter server lifecycle via CancellationToken.
    /// Created in setup(), stored in AppState, cancelled on RunEvent::Exit.
    pub struct AdapterLifecycle {
        cancel_token: CancellationToken,
    }

    impl AdapterLifecycle {
        pub fn new() -> Self;
        /// Get a child token to pass to axum::serve with_graceful_shutdown
        pub fn shutdown_token(&self) -> CancellationToken;
        /// Cancel the token — triggers graceful shutdown of axum server
        pub fn shutdown(&self);
    }
    ```

    The `CancellationToken` is `Clone + Send + Sync`. The `shutdown_token()` returns a child token
    so that `shutdown()` cancels the child chain. The parent token is stored in AppState for later
    cancellation by the lifecycle callback.
  </action>

  <name>Task 8: Update lib.rs — wire adapter server startup and shutdown</name>
  <files>apps/desktop/src-tauri/src/lib.rs</files>
  <action>
    Wire the adapter server into the Tauri lifecycle.

    Changes needed:

    1. **Add `mod adapter;`** module declaration (alphabetically before `mod commands;`)

    2. **Setup: Start adapter server**
       In the setup closure, after tray initialization:
       ```rust
       // Start adapter HTTP server
       let handle = app.handle().clone();
       let lifecycle = adapter::lifecycle::AdapterLifecycle::new();
       let shutdown_token = lifecycle.shutdown_token();

       // Store lifecycle in AppState
       {
           let state = app.state::<Mutex<state::AppState>>();
           if let Ok(mut guard) = state.lock() {
               *guard.adapter_lifecycle.lock().unwrap() = Some(lifecycle);
           }
       }

       // Spawn server on shared Tokio runtime
       tauri::async_runtime::spawn(async move {
           let port = adapter::start_server(handle, shutdown_token).await;
           tracing::info!("Adapter server started on port {}", port);
       });
       ```

    3. **Change app lifecycle: Split .run() into .build() + .run(event)**
       Replace:
       ```rust
       .run(tauri::generate_context!())
       .expect("error while running tauri application");
       ```
       With:
       ```rust
       .build(tauri::generate_context!())
       .expect("error while building tauri application")
       .run(|app_handle, event| {
           if let tauri::RunEvent::Exit = event {
               // Graceful shutdown of adapter server
               let state = app_handle.state::<Mutex<state::AppState>>();
               if let Ok(guard) = state.lock() {
                   if let Ok(mut lifecycle_guard) = guard.adapter_lifecycle.lock() {
                       if let Some(lifecycle) = lifecycle_guard.as_ref() {
                           lifecycle.shutdown();
                           tracing::info!("Adapter server shut down gracefully");
                       }
                   }
               }
           }
       })
       .expect("error while running tauri application");
       ```

    Note: The `App` returned by `.build()` must be consumed by `.run()`. Make sure the previous
    `.expect()` on `.run()` is moved to the `.build()` call.
  </action>
